ภาพรวม

เมนูคอร์สใช้รูปแบบ Master–Detail: หัวข้อหลัก (Topic) เรียงเป็นเมนูฝั่งซ้าย เมื่อคลิกหัวข้อใด รายการคอร์สของหัวข้อนั้นจะแสดงในพื้นที่รายละเอียดฝั่งขวา

ปัจจุบันรายการ “คอร์สอื่นๆ & บริการ” (FORWARD, Digital SAT MATH, สินค้าพรีเมียม, คอร์สบัญชี, ทดลองเรียน) ถูกวางเป็น sub-group ซ้อนอยู่ภายใต้ Topic “คอร์สเรียนรอบสด” ผู้ใช้จึงต้องเข้าหัวข้อรอบสดก่อนถึงจะเจอบริการเหล่านี้ ทั้งที่เนื้อหาไม่เกี่ยวกับ “คอร์สสด” โดยตรง โดยเฉพาะ “ทดลองเรียนฟรี” ที่เป็นจุด conversion สำคัญแต่ถูกฝังลึกที่สุด

การเปลี่ยนแปลงใน v2: ยก “คอร์สอื่นๆ & บริการ” ขึ้นเป็น Topic หลักลำดับที่ 5 และยก “ทดลองเรียนฟรี” แยกออกมาเป็น Topic หลักลำดับที่ 6 ระดับเดียวกับหัวข้ออื่น (ไม่ซ้อนใต้ Topic 5 อีก) และให้ “คอร์สเรียนรอบสด” เหลือเฉพาะคอร์สสดจริง ๆ
2

ปัญหา & เป้าหมาย

  • รายการบริการสำคัญ (เช่น ทดลองเรียนฟรี, FORWARD) ถูกฝังลึกใต้หัวข้อที่ไม่เกี่ยวข้อง ทำให้ค้นเจอยาก (discoverability ต่ำ)
  • ชื่อ “คอร์สเรียนรอบสด” สื่อความหมายไม่ตรง เพราะมีของที่ไม่ใช่คอร์สสดปนอยู่
  • Taxonomy ไม่สม่ำเสมอ — บาง Topic ปนเนื้อหาคนละประเภท

เป้าหมายที่วัดผลได้

  • เพิ่มอัตราการคลิกเข้าดูคอร์สเรียนทุกประเภท ทดสอบได้โดยเปรียบเทียบกับ New users เทียบ Conversion rate ของ 2 Design
  • ลดความสับสนของ taxonomy (หัวข้อย่อย) โดยให้แต่ละ Topic มีเนื้อหาประเภทเดียวกัน

โครงสร้างข้อมูล (Data model)

เมนูทั้งหมดขับเคลื่อนด้วย config array ชุดเดียว — แก้คอร์ส = แก้ข้อมูล ไม่ต้องแตะ layout

// 'NEW' = badge สีฟ้า, ค่าอื่น (เช่น 'V.เต็ม') = สไตล์ live/amber
type Badge = 'NEW' | string;

interface CourseItem {
  label: string;
  href: string;
  badge?: Badge;
}
interface CourseGroup {
  label: string;
  items: CourseItem[];
}
interface CourseTopic {
  id: string;        // slug คงที่ ใช้ใน analytics/anchor
  title: string;
  icon: string;      // ชื่อ icon (เช่น 'ti-package')
  isNew?: boolean;   // แสดง tag "ใหม่" บนเมนูซ้าย
  groups: CourseGroup[];
}

ตัวอย่าง Topic ใหม่ (ลำดับที่ 5 และ 6)

พฤติกรรม UI (Interaction)

สถานะพฤติกรรม
Defaultเปิด Topic แรก (tcas) เป็น active โดยอัตโนมัติ
Hover เมนูซ้ายพื้นหลังเปลี่ยนเป็นสีขาว, cursor pointer
Active เมนูซ้ายพื้นหลังขาว + icon และ label เป็นสีแบรนด์
คลิก Topicพื้นที่ขวาอัปเดตทันที (ไม่ reload), เปลี่ยน aria-selected
Hover รายการคอร์สสีข้อความเปลี่ยนเป็น brand-ink
คลิกรายการคอร์สนำทางไป href ของคอร์สนั้น
Tag “ใหม่”แสดงเฉพาะ Topic ที่ isNew === true (เฟสนี้คือ Topic 5 และ Topic 6)

Badge logic

  • badge === 'NEW' → pill สีฟ้า
  • badge มีค่าอื่น (เช่น V.เต็ม, V.รวบรัด, ฟรี) → pill สีเหลือง
  • ไม่มี badge → ไม่แสดง

Responsive — Mobile First

ออกแบบและพัฒนาจาก mobile เป็นค่าเริ่มต้น (base) แล้วค่อยขยายขึ้น desktop ด้วย @media (min-width: …) — ไม่ใช่เขียน desktop ก่อนแล้วลดลง

BreakpointLayout
Base · mobile (เริ่มที่ ~390px)คอลัมน์เดียว: rail เป็นแท็บแนวนอน scroll ได้ แสดงเฉพาะ icon (ป้าย “ใหม่” เป็น badge เล็กมุม icon), detail อยู่ด้านล่าง, รายการคอร์สเรียง 1 คอลัมน์
min-width: 761px (tablet/desktop)ขยายเป็น Master–Detail 2 คอลัมน์: rail ซ้าย 248px แสดง icon + label เต็ม, detail ขวา, รายการคอร์สหลายคอลัมน์

หลักการ

  • Base CSS = layout mobile ทั้งหมด ไม่ต้องพึ่ง media query — จอเล็กสุดต้องใช้งานได้ครบ
  • Touch target ≥ 44×44px สำหรับแท็บ icon และรายการคอร์สบน mobile
  • รายการในแต่ละ group ใช้ grid auto-fit minmax(190px, 1fr) ที่ breakpoint desktop — จำนวนคอลัมน์ปรับตามความกว้างเอง
  • เนื้อหา/ลำดับ Topic เหมือนกันทุกขนาดจอ — ต่างกันเฉพาะ layout
8

States & edge cases

  • Topic ไม่มีคอร์ส — ถ้า items ว่าง แสดงข้อความ “ยังไม่มีคอร์สในหมวดนี้” แทนพื้นที่ว่าง
  • Group เดียว — ถ้ามีกลุ่มเดียวและชื่อซ้ำกับ Topic อาจไม่ต้องแสดง group label ซ้ำ (พิจารณาตามดีไซน์)
  • ป้าย “ใหม่” — ควรมีวันหมดอายุ (เช่น 30–60 วันหลังเพิ่ม) ไม่ค้างถาวร
  • Keyboard — เมนูซ้าย focusable, เลือกด้วย Enter/Space, มี visible focus ring
  • Analytics — ยิง event ตอนคลิก Topic และคลิกคอร์ส โดยใช้ topic.id + item.label
